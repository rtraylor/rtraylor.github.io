A Partition by any Other Name https://www.themathcitadel.com/a-partition-by-any-other-name/ Tue, 14 Nov 2017 19:30:42 +0000 admin http://www.themathcitadel.com/?p=1526 I promise I'm actually a probability theorist, despite many of my posts being algebraic in nature. Algebra, as we've seen in several other posts, elegantly generalizes many things in basic arithmetic, leading to highly lucrative applications in coding theory and data protection.  Some definitions in mathematics may not have obvious "practical use", but turn out to yield theorems and results so powerful we can use them to send image data cleanly from space.  This is a continuation of the exploration of the algebra behind many aspects of coding theory and data protection, and we will continue to develop the fundamentals necessary to appreciate the elegance and simplicity of code transmission and data protection. Here we will discuss the notion of a coset as a subgroup of a finite group [latex]G[/latex]. For a discussion on the definition of a group, please check out this post. Recall that a group is a set [latex]G[/latex] combined with an operation we will denote [latex]\cdot[/latex] that satisfies the following axioms: (1) Closure under [latex]\cdot[/latex]: If [latex]a[/latex] and [latex]b[/latex] are in [latex]G[/latex], then their product (or sum)[note]Remember that operations aren't just the typical addition or multiplication you use every day. We use addition and multiplication as general operations, and the product is just the name for one element "operation" another.[/note] [latex]a\cdot b[/latex] is in [latex]G[/latex] (2) Associativity: Let [latex]a,b[/latex] and [latex]c[/latex] be elements of [latex]G[/latex]. Then we can group the operation however we like. That is, [latex, display = true] a\cdot (b\cdot c) = (a\cdot b)\cdot c[/latex] (3) Existence of an identity element: There must be some element [latex]e[/latex] in [latex]G[/latex] that, when multiplied on either the left or right by any element [latex]a[/latex] in [latex]G[/latex], returns that element. Mathematically, there exists [latex]e \in G[/latex] such that for any [latex]a \in G[/latex] [latex, display = true]e\cdot a = a\cdot e = a[/latex] (4) Existence of an inverse: For each element [latex]a[/latex] in [latex]G[/latex], there is another element [latex]a^{-1}[/latex] in [latex]G[/latex] such that multiplying [latex]a^{-1}[/latex] on either the left or right by [latex]a[/latex] yields the identity element [latex]e[/latex]. Formally, for each [latex] a \in G[/latex] there exists [latex]a^{-1}[/latex] in G such that [latex, display = true] a\cdot a^{-1} = a^{-1}\cdot a = e[/latex] Some quick examples of groups:
  • [latex](\mathbb{R}, +)[/latex], all real numbers under regular addition. The identity element is 0, and the inverse of any real number [latex]a[/latex] is [latex]-a[/latex].
  • [latex]\mathbb{Z}_{n}[/latex], integers modulo [latex]n[/latex] under modulo addition. 
  • [latex]n \times n[/latex] matrices under matrix addition
The number of elements in a group [latex]G[/latex] can be either finite or infinite and is denoted [latex]|G|[/latex]. In [latex]\mathbb{Z}_{3} = \{0,1,2\}[/latex], [latex]|\mathbb{Z}_{3}| = 3[/latex]. [latex]|\mathbb{R}|[/latex] is infinite[note]There are different kinds of infinite, and different sizes of infinity. Cardinality of infinite sets gets weird, and we'll explore this later.[/note]. We're going to stick with finite groups; that is, groups with a finite number of elements. 

Subgroups

A subset [latex]T[/latex] of a set [latex]S[/latex] is simply a set made from some of the elements in [latex]S[/latex]. If [latex]S = \{0,1,2,3,4,5\}[/latex], then [latex]T = \{0,2,4\}[/latex] is a subset of [latex]S[/latex], and we denote it [latex]T \subseteq S[/latex]. The even integers are a subset of all integers. The rational numbers area subset of all real numbers. The set of square matrices is a subset of all matrices.  If our set has an operation and is actually a group [latex]G[/latex], then a subset of this group that also fits the definition of a group is called a subgroup, and we write [latex]H \leq G[/latex]. Not all subsets of a group are subgroups. To determine if a subset of a group is actually a subgroup, we just have to verify properties (1) and (4) above for the subset.  Let's focus on finite groups for now. Obviously, we can have subgroups of infinite groups, but it's easier to get used to these ideas on finite groups first. One possible way to construct a subgroup of a group is to pick an element [latex]a \in G[/latex]. We can take powers of this element and collect them together in a nice bag to create a subgroup called [latex]H[/latex], or for fun, denote it [latex]\langle a \rangle[/latex] [latex]H = \langle a\rangle = \{a, a^{2} = a\cdot a, a^{3} = a\cdot a \cdot a,\ldots \}[/latex] Now, are we ever done taking powers of [latex]a[/latex]? Actually, yes. In a finite group, there are only a finite number of elements total, so we have to stop taking powers at some point. Moreover, we'll get to the point where some power of [latex]a[/latex] actually gives us the identity element [latex]e[/latex], and then any further powers just start us over. Let's take an explicit example.  Suppose we take [latex]\mathbb{Z}_{6} = \{0,1,2,3,4,5\}[/latex] under modulo addition. Pick 2. Now let's see what happens when we create a subgroup from powers of 2 under modulo 6 addition.  First, we just have [latex]2[/latex]. Then [latex]2^{2} = (2+ 2) \bmod 6 = 4[/latex][note]Remember that the operation is addition, so 2 "operation" 2  is 2+2 modulo 6[/note]. Next, [latex]2^{3} = (2+2+2) \bmod 6 = 6 \bmod 6 \equiv 0[/latex]. Hey look! Our identity element. That means that [latex]2^{4} = 2^{3}\cdot 2 = 0 + 2 = 2[/latex]. Now we started over. That means there are only 3 distinct powers of 2 under modulo 6 addition: [latex]H = \langle 2\rangle = \{0,2,4\}[/latex]. This is the subgroup generated by 2, which is why we use the notation [latex]\langle 2\rangle[/latex], because 2 and its powers under this group operation constructs the subgroup. We call [latex]|\langle a \rangle |[/latex] the order of [latex]a[/latex]. Put another way, the order of an element in a group is the power required to yield the identity element. So, under modulo 6, [latex]2^{3} \equiv 0[/latex], so the order of 2 is 3, which is the size of the subgroup generated by 2.

Coset Decomposition using Subgroups

We can take a subgroup of a finite group [latex]G[/latex] and partition it relative to that subgroup in a nice, methodical way[note]This only works for finite groups[/note]. Let's take a subgroup generated by some element [latex]h \in G[/latex]. That is, let our subgroup [latex]H = \langle h \rangle = \{h_{1}, h_{2},...,h_{c} = e\}[/latex] where [latex]c[/latex] is the order of [latex]h[/latex]. We know the order of the element must be finite, because the group itself is finite, and all groups are closed under operations. Now let's construct a pretty array: First, we'll put the elements of our subgroup [latex]H[/latex] on the first row: [latex, display = true]\begin{array}{ccccc}h_{1}&h_{2}&h_{3}&\ldots&h_{c}\end{array}[/latex] Now, pick any [latex]g \in G[/latex] you want that is not in the subgroup [latex]H[/latex]. Let's name it [latex]g_{2}[/latex]. Put it at the start of the second row, and each element of the second row is formed by multiplying this [latex]g_{2}[/latex] on the left by the column header [latex]h_{i}[/latex]: [latex, display = true]\begin{array}{ccccc}h_{1}&h_{2}&h_{3}&\ldots&h_{c}\\g_{2}&g_{2}\cdot h_{2}& g_{2} \cdot h_{3}&\ldots&g_{2} \cdot h_{n}\end{array}[/latex] That was easy enough. Now pick another [latex]g_{3}\in G[/latex] that hasn't shown up in the first two rows. Then we'll create the third row in the same way as we created the second: [latex, display = true]\begin{array}{ccccc}h_{1}&h_{2}&h_{3}&\ldots&h_{c}\\g_{2}&g_{2}\cdot h_{2}& g_{2} \cdot h_{3}&\ldots&g_{2} \cdot h_{n}\\g_{3}&g_{3}\cdot h_{2}& g_{3} \cdot h_{3}&\ldots&g_{3} \cdot h_{n}\end{array}[/latex] We may continue making the array in this way until all of our elements are used up. Then we end up with the elements of our group arranged nicely into an [latex]m \times n[/latex] array: [latex, display = true]\begin{array}{ccccc}h_{1}&h_{2}&h_{3}&\ldots&h_{c}\\g_{2}&g_{2}\cdot h_{2}& g_{2} \cdot h_{3}&\ldots&g_{2} \cdot h_{n}\\g_{3}&g_{3}\cdot h_{2}& g_{3} \cdot h_{3}&\ldots&g_{3} \cdot h_{n}\\\vdots&&&&\\g_{m}&g_{m}\cdot h_{2}& g_{m} \cdot h_{3}&\ldots&g_{m} \cdot h_{n}\end{array}[/latex] Each row in this array forms what we call a left coset of [latex]H[/latex] in [latex]G[/latex]. You take an element of the group and multiply each element of the subgroup on the left by that element. Notice that for a finite group, we have [latex]m[/latex] left cosets. If we had multiplied those [latex]g_{j}[/latex]'s on the right, we would call the rows right cosets. If the operation is commutative ([latex]a\cdot b = b\cdot a[/latex]), then left and right cosets are the same thing.  We can prove that every element in a finite group shows up in this array, and that no element appears more than once in the array[note]I'm not going to prove this here. This proof is done by contradiction, and is useful and necessary, but not particularly illuminating to this discussion[/note]. What we've actually done is partitioned a finite group into these rows using our subgroup [latex]H[/latex]. This partitioning is called a coset decomposition.  (Remark. In fact, a coset on a finite group is a type of equivalence relation. This very elegant fact will play a large role in coding theory syndromes in a future post. For a discussion of equivalence relations please see the video below:) [embedyt] https://www.youtube.com/watch?v=ELVrHyPiI9c[/embedyt]

LaGrange's Theorem

For finite groups, this coset decomposition yields LaGrange's theorem, a particularly beautiful theorem that relates the size of a subgroup, the number of cosets of that subgroup, and the size of a group. Because mathematicians hate writing more than we have to, let's write the number of left (or right) cosets of [latex]H[/latex] in [latex]G[/latex] as [latex][G:H][/latex].
Lagrange's Theorem Suppose [latex]|G|[/latex] is finite, and [latex]H \leq G[/latex][note]Remember, that's our notation for "H is a subgroup of G"[/note]. Then  [latex, display = true]|G| = [G:H] |H|[/latex]
The number of cosets with respect to [latex]H[/latex] times the size of [latex]H[/latex] gives us the size of the group. Take a look back at our array we created. That's a visual illustration of this theorem. The number of rows gives us the number of cosets, and the number of columns is the size of our subgroup [latex]H[/latex]. All group elements appeared in the array, so we can see that the number of cosets times the size of the subgroup gives us the size of the original group [latex]G[/latex].

Conclusion

Cosets exist for infinite groups as well, though they cannot be constructed quite the same way we did above, since we would never "finish" filling up rows and columns. For those, we just create them by multiplying some element [latex] g \in G[/latex] that isn't in a subgroup [latex]H[/latex] (which may not be finite anymore) on the left or right by all elements in the subgroup. We write a general left coset as  [latex, display = true] gH = \{gh : h \in H\}[/latex] The product of that [latex]g[/latex] with all elements of the subgroup [latex]H[/latex]. If the group is infinite, we don't necessarily get a coset decomposition like we had before, and LaGrange's Theorem doesn't apply anymore.  Sticking with finite groups can yield some extremely powerful uses though. We'll need all of this (and some linear algebra) when we discuss how to store and decode a binary code that may be transmitted over a noisy channel. The fact that the coset decomposition yields a set of equivalence classes will allow us to drastically cut the amount of bits we need to store to fully represent a binary code and its parity check equations. This will allow us to use very large block length for code transmission and data protection, as we will see in future posts.

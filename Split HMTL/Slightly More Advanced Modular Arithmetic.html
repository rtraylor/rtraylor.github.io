Previously, I've written a couple articles discussing modular arithmetic that visited the basics of what exactly happens when we move to "clockwork" type arithmetic. This article will cover some slightly more advanced notions and bring in a little basic number theory.

Brief Review

When we partition the integers modulo some natural number [latex]n[/latex], we're creating equivalence classes to "bin" the integers by what they give as a remainder when divided by [latex]n[/latex]. We denote this set of equivalence classes [latex]\mathbb{Z}\setminus n\mathbb{Z}[/latex] or [latex]\mathbb{Z}_{n}[/latex].

[latex, display = true]\mathbb{Z}_{n} = \{0,1,\ldots, n-1\}[/latex]

As an explicit example, take [latex]\mathbb{Z}\setminus 3\mathbb{Z}[/latex]. When we divide any integer by [latex]3[/latex], we will get a remainder of [latex]0,1[/latex] or [latex]2[/latex]. Those are our only options. If we bucket all the integers by what remainder we get when we divide by 3, we get three equivalence classes we'll denote [latex]\bar{0}, \bar{1}, \bar{2}[/latex]. 0,1, and 2 are representatives of each of these three classes, and the main ones we work with since it's the easiest. When we write the equivalent class, we use the notation

[latex,display = true] x \equiv y \mod n[/latex]

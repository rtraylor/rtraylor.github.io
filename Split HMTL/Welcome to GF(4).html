Welcome to GF(4) https://www.themathcitadel.com/welcome-to-gf4/ Sun, 26 Nov 2017 19:54:49 +0000 admin http://www.themathcitadel.com/?p=1595 Everyone has solved some version of a linear system in either high school or college mathematics. If you've been keeping up with some of my other posts on algebra, you know that I'm about to either take something familiar away, or twist it into a different form. This time is no different; we're going to change the field we operate over, and solve a basic linear system in a Galois field called GF(4).

Linear Systems

Let's start with an example of a regular, simple linear system in two variables: [latex, display = true] \begin{aligned}2x+y&=3\\x+2y&=3\end{aligned}[/latex] The goal here is to figure out all the possible [latex](x,y)[/latex] pairs that satisfy both of these equations at the same time. Both high school and college algebra teach several different methods to solve these types of equations. We can use substitution-- solving perhaps the top equation for [latex]y[/latex] to express [latex]y[/latex] in terms of [latex]x[/latex], then substituting that expressing in the second equation to get an equation with all [latex]x[/latex]. From there, we know how to solve for [latex]x[/latex]. When we get a numeric answer, we can plug it back into our expression for [latex]y[/latex] and get our full solution. Let's solve the above system this way just to refresh. First, solve the top equation for [latex]y[/latex] [latex, display = true]y = 3-2x[/latex] Great. Now substitute the right-hand side of this equation wherever we see [latex]y[/latex] in the second equation: [latex, display = true]x + 2(3-2x) = 3[/latex] Now, solve this equation for [latex]x[/latex] to get [latex]x=1[/latex]. Substitute this back and get [latex]y = 3-2 = 1[/latex]. 

Matrix form - a different way to express

We can actually express this equation in matrix form. This will look like [latex]A\mathbf{x} = \mathbf{v}[/latex], where [latex]\mathbf{x}[/latex] and [latex]\mathbf{v}[/latex] are vectors. The coefficients of the system of equations form the matrix [latex]A[/latex]. The variables we want to solve for are the vector [latex]\mathbf{x}[/latex], and the vector [latex]\mathbf{v}[/latex] represents the right hand sides of the equations. Our system in matrix form is  [latex, display = true]\begin{bmatrix}2&1\\1&2\end{bmatrix}\begin{bmatrix}x\\y\end{bmatrix} = \begin{bmatrix}3\\3\end{bmatrix}[/latex] Multiplying the left-hand side out will return the original form we saw earlier. This is simply a different way to express the same system of equations. We typically like to use matrix form because arithmetic and algebra on matrices yields many useful properties and methods for solving systems much larger than our example here.[note]I doubt anyone wants to solve a linear system in 100 variables by substitution, for example.[/note]

Solving linear systems in matrix form

There are several different methods for solving linear systems once we have them in matrix form. Linear algebra courses will discuss a technique called row reduction, which is incredibly useful. We won't discuss that one in this post. We're going to use a really handy rule called Cramer's rule. Basically, if we take a linear system with as many equations as we have variables, and a unique solution exists[note]A linear system may have one of three possible options: no solution, one solution, or infinitely many solutions. Because this isn't really a post on linear algebra per se, we're going to safely assume that anything I throw at you in this post will have a unique solution.[/note], then we actually have an explicit formula that gives us the numerical answer for each variable in our system. We'll quickly discuss how to compute a determinant of a [latex]2\times 2[/latex] matrix so we can use Cramer's rule.[note]Matrix theory and determinants deserve their own study for sure. The point of this post isn't really to dive into those. Right now, determinants are just computational tools for us. It's ok to just use a tool and get used to it before we dive into how it works.[/note]

Computing a [latex]2\times 2[/latex] determinant

The entries of a matrix [latex]A[/latex] are written as [latex]a_{ij}[/latex], where [latex]i[/latex] tells you the row it's in, and [latex]j[/latex] tells you the column. A general [latex]2\times 2[/latex] matrix looks like this: [latex, display = true]A = \begin{bmatrix}a_{11} & a_{12}\\a_{21}&a_{22}\end{bmatrix}[/latex] The determinant  of a [latex]2\times 2[/latex] matrix is a number, and is calculated as[note]The theory of exactly what a determinant is and does can get quite deep and require a bit of legwork to get there. I'm sidestepping it on purpose, because it's not illustrative for this post.[/note] [latex]\text{det}(A) = a_{11}\cdot a_{22} - a_{12}\cdot a_{21}[/latex] Back to our linear system, [latex]\text{det}(A) = 2\cdot 2 - 1\cdot 1 = 4-1=3[/latex].

Back to Cramer's rule

Now, the two variables we need to solve for are [latex]x[/latex] and [latex]y[/latex]. We put them in that order for a reason. The right-hand side was given as the vector [latex]\mathbb{v} =\begin{bmatrix}3\\3\end{bmatrix}[/latex]. We're going to substitute this column in to either the first or second column of [latex]A[/latex] when we're computing the answer for [latex]x[/latex] and [latex]y[/latex], respectively.  Let's say we're focusing on [latex]x[/latex], the first variable. Put that [latex]\mathbb{v}[/latex] where the first column of [latex]A[/latex] was. Then we'll get a new matrix  [latex, display = true]A_{x} = \begin{bmatrix}3 & 1\\ 3 & 2\end{bmatrix}[/latex] Let's do the same for [latex]y[/latex], only this time we'll put [latex]\mathbb{v}[/latex] in for the 2nd column, since it's our 2nd variable.  [latex, display = true]A_{y} = \begin{bmatrix}2 & 3\\1&3\end{bmatrix}[/latex] Now, Cramer's rule tells us that  [latex, display = true]x = \text{det}(A)^{-1}\cdot\text{det}(A_{x}) \text{ and } y =\text{det}(A)^{-1}\cdot\text{det}(A_{y})[/latex] Nice and easy. We already know how to compute the determinant of a [latex]2\times 2[/latex] matrix, so we're basically done. Now, I used very particular notation. I wrote [latex]\text{det}(A)^{-1}[/latex] instead of [latex]\frac{1}{\text{det}(A)}[/latex] on purpose. I wanted to indicate that it is the multiplicative inverse of the number that is [latex]\text{det}(A)[/latex]. We have to be careful when discussing multiplication and division, because division as an operation doesn't always exist. This will be important when I twist around addition and multiplication soon.  Using our newfound Cramer's rule, we can solve our equation this way. The multiplicative inverse of 3 is the fraction 1/3, because multiplying these two together gives us 1, the multiplicative identity of the real numbers. [latex, display = true]x = \frac{1}{3}\cdot \text{det}(A_{x}) = \frac{1}{3}\cdot 3 = 1[/latex] [latex, display = true]y = \frac{1}{3}\cdot\text{det}(A_{y}) = \frac{1}{3}\cdot 3 = 1[/latex] And we have our answer again, just obtained in a different way. 

GF(4): our first sighting of a Galois field

We've seen groups before in several posts. We know that a group is a set of things combined with an operation that satisfies certain properties. I'm no longer satisfied with having just one operation on a set anymore. I want two operations - I want addition and multiplication.[note]Addition and multiplication are just the things we're going to name our two operations. They don't have to mean exactly the same thing as addition and multiplication on real numbers, as we'll see shortly.[/note] Now, we'll call these two operations [latex]+[/latex] and [latex]\cdot[/latex]. I'm going to jump ahead a bit of building groups into rings then fields and just go ahead and define a field:
A field is a set [latex]F[/latex] together with two operations [latex]+[/latex] and [latex]\cdot[/latex] that satisfy the following properties: (1) [latex](F,+)[/latex] is an abelian group.[note]A group where a+b = b+a[/note] (2) [latex]F[/latex] is closed under multiplication.[note]The product of two elements doesn't leave the set F[/note] (3) The nonzero[note]0 is the identity element for the operation +[/note] elements of [latex]F[/latex] form an abelian group under [latex]\cdot[/latex] (4) We get the distributive law: [latex](a+b)\cdot c = (a\cdot c) + (b\cdot c)[/latex]
We just added an operation and some requirements to make sure nothing too weird happens. We can create addition and multiplication tables just like we did here. We're going to take a look at an example of a very specific field called a Galois field. This is simply a finite field with [latex]q[/latex] elements, if it exists.[note]Fun fact: Galois fields only exist if the number of elements is a prime or a power of a prime. So there is no Galois field with 6 elements.[/note] We denote a Galois field with [latex]q[/latex] elements [latex]\text{GF}(q)[/latex].  The field [latex]\text{GF}(4)[/latex] has addition and multiplication tables that look like this.[note]Right now, just work with them as given. There is a way to generate the arithmetic over Galois fields in general, but we'll begin tackling that in later posts. We just want to get used to the idea that numbers don't add and multiply the same way anymore. They're just symbols now. 2+3 in GF(4) doesn't have a tactile interpretation like "2 things and three more things".[/note] [latex, display = true]\begin{array}{l|rrrr}\textcolor{#893B2F}{+}&\textcolor{#893B2F}{0}&\textcolor{#893B2F}{1}&\textcolor{#893B2F}{2}&\textcolor{#893B2F}{3}\\\textcolor{#893B2F}{0}&0&1&2&3\\\textcolor{#893B2F}{1}&1&0&3&2\\\textcolor{#893B2F}{2}&2&3&0&1\\\textcolor{#893B2F}{3}&3&2&1&0\end{array}\qquad\begin{array}{l|rrrr}\textcolor{#893B2F}{\cdot}&\textcolor{#893B2F}{0}&\textcolor{#893B2F}{1}&\textcolor{#893B2F}{2}&\textcolor{#893B2F}{3}\\\textcolor{#893B2F}{0}&0&0&0&0\\\textcolor{#893B2F}{1}&0&1&2&3\\\textcolor{#893B2F}{2}&0&2&3&1\\\textcolor{#893B2F}{3}&0&3&1&2\end{array}[/latex] Let's note that the additive identity is indeed the "number" 0.[note]When you add it to anything, you get that anything back.[/note] The multiplicative identity is the "number" 1.[note]Again, multiplying it by anything gives the anything back.[/note] Now, spend some time getting comfortable with this table and let's answer a few questions:

(1) What's the additive inverse of each element?

For each of 0, 1, 2, and 3, which element returns the additive identity 0 when we add it to 0, 1, 2, and 3?
  • 0 is clearly its own additive inverse. 0+0 = 0
  • For 1, which element added to 1 returns 0? Looking at the addition table, we see that 1+1 = 0. Thus, 1 is its own additive inverse. Another way to write it is that -1 = 1, where [latex]-x[/latex] denotes the additive inverse of the element [latex]x[/latex] and not the number [latex]-1[/latex] multiplied by [latex]x[/latex]
  • Continuing this exercise, we find that each element of GF(4) is its own additive inverse.

(2) What's the multiplicative inverse of each element?

We'll repeat the same exercise as in (1), except using the multiplication table. The multiplicative identity is 1, so to find the multiplicative inverse of each element [latex]x[/latex], we find another element [latex]y[/latex] such that [latex]x\cdot y = y\cdot x = 1[/latex]. We can read this off our nice multiplication table:
  • 0 has no multiplicative inverse. Uh oh! Is GF(4) broken? No. Recall part (3) of the field definition above. The elements excluding 0 must form an abelian group under multiplication. That means it's totally ok for 0 times everything to return 0, and for 0 to have no multiplicative inverse.[note]We did this on purpose. If we didn't have this definition, the real numbers wouldn't be a field either. When generalizing mathematics, we don't want to break what we already have.[/note]
  • 1 is its own multiplicative inverse. Makes sense. It's the multiplicative identity.
  • The multiplicative inverse of 2 is 3 from the table, so the multiplicative inverse of 3 is 2. [latex]2\cdot 3 = 3\cdot 2 = 1[/latex].
This is fine. We're defining it this way, and we are allowed to do that, provided anything we define fits the definition of a field. We aren't in the real numbers anymore, friends. The symbols look like real numbers, but they no longer act like real numbers. When we are in GF(4), this is how these numbers behave under addition and multiplication as we define it for GF(4). 

Solving our earlier system, but now in GF(4)

Let's return to our earlier system of equations that we solved using Cramer's rule. The equations look the same, but will we get the same solution if we solve it over GF(4), with our new addition and multiplication tables? Here's our equation again in matrix form. [latex, display = true]\begin{bmatrix}2&1\\1&2\end{bmatrix}\begin{bmatrix}x\\y\end{bmatrix} = \begin{bmatrix}3\\3\end{bmatrix}[/latex] [latex]A_{x}[/latex] and [latex]A_{y}[/latex] are also the same.  [latex, display = true]A_{x}=\begin{bmatrix}3&1\\3&2\end{bmatrix}~\qquad~A_{y}=\begin{bmatrix}2&3\\1&3\end{bmatrix}[/latex] To apply Cramer's rule, we need the determinant of [latex]A[/latex], [latex]A_{x}[/latex], and [latex]A_{y}[/latex] under the rules of arithmetic in GF(4). The method of computing a [latex]2\times 2[/latex] determinant has not changed, but our answers will.

Compute [latex]\text{det}(A)[/latex], [latex]\text{det}(A_{x})[/latex], and [latex]\text{det}(A_{y})[/latex]

[latex, display = true]\text{det}(A)=2\cdot 2-1\cdot 1[/latex] Be careful here, and let's use our addition and multiplication tables for reference. [latex]2\cdot 2 = 3[/latex] now, and [latex]1\cdot 1 = 1[/latex]. So [latex, display = true]\text{det}(A)=2\cdot 2-1\cdot 1=3-1[/latex] We need to read carefully again here too. [latex]a-b[/latex] in abstract algebra is [latex]a[/latex] plus the additive inverse of [latex]b[/latex], which isn't the number [latex]-a[/latex] here. Remember, in GF(4) we only get 4 elements: 0, 1, 2, and 3. No negative numbers here. They don't exist. Check above to note that the additive inverse of 1 is 1, so [latex, display = true]\text{det}(A)=2\cdot 2-1\cdot 1=3-1=3+1=2[/latex] where 3+1 can be read from the addition table for GF(4).  Pause. Let that sink in. We've twisted arithmetic a bit from what you're used to with real numbers. There were some properties you took for granted, like the existence of negative numbers. The only tools we have are those two group tables above. That tells us everything we need to know to do arithmetic over GF(4).  Let's do this again and find the determinants of [latex]A_{x}[/latex] and [latex]A_{y}[/latex], respectively in the same way we did [latex]\text{det}(A)[/latex] [latex, display = true]\text{det}(A_{x})=3\cdot 2-3\cdot 1=1-3=1+3=2[/latex] (Remember that [latex]1-3[/latex] is [latex]1[/latex] plus the additive inverse of [latex]3[/latex], which is [latex]3[/latex], so [latex]1-3=1+3[/latex]) [latex, display = true]\text{det}(A_{y})=2\cdot 3-3\cdot 1=1-3=1+3=2[/latex] Again, remember that [latex]1-3[/latex] is [latex]1[/latex] plus the additive inverse of [latex]3[/latex]. [latex]3[/latex] is its own additive inverse, so [latex]1-3=1+3=2[/latex].

Find the solution

We're almost done; we have all the pieces. Cramer's rule tells us that  [latex, display = true]x=\text{det}(A)^{-1}\cdot\text{det}(A_{x})\text{ and }y =\text{det}(A)^{-1}\cdot\text{det}(A_{y})[/latex] See, there was a reason I wrote it the way I did. There are no fractions in GF(4); fractions are the multiplicative inverses of real numbers. I wrote it in the more general way so we could see that. The multiplicative inverse of [latex]\text{det}(A)=2[/latex] can be read off the multiplication table for GF(4): [latex]\text{det}(A)^{-1}=3[/latex], because [latex]2\cdot 3[/latex] gives us the multiplicative identity [latex]1[/latex].  Finally then, we're down to simple multiplication to get our new solutions in GF(4): [latex, display = true]x=\text{det}(A)^{-1}\cdot\text{det}(A_{x})=3\cdot 2=1[/latex] [latex, display = true]y=\text{det}(A)^{-1}\cdot\text{det}(A_{y})=3\cdot 2=1[/latex]

Conclusion

Notice that this solution is the same as when we lived in our comfortable world of real numbers, but this is a total coincidence. The equations were the same, the numbers involved were the same, but we changed what addition and multiplication did by moving to a new field called GF(4). The purpose of this exercise was to get used to the idea of arithmetic in a new space, and to see what an example of a Galois field looks like. Explaining how to generate these Galois fields in general and defining their addition and multiplication tables will get a bit involved; we'll tackle these soon. For now, it's important just to let go of our tightly held arithmetic notions that are really special properties of real numbers.
